<h1>Sales Data Analysis Capstone Project with Python and Quarto</h1>
<p>Welcome to the Sales Analysis Capstone Project! This project is a beginner-friendly, hands-on walkthrough of data science tasks aimed at uncovering retail sales insights. It is designed to help users leverage Python and Quarto for data analysis, visualization, and reporting.</p>
<h2>Goals:</h2>
<ul>
<li>Explore retail sales data and derive actionable insights</li>
<li>Analyze seasonal and trend impacts on sales</li>
<li>Identify city and district-based sales patterns</li>
<li>Understand sales behaviors during high-traffic events (e.g., Black Friday, Cyber Monday, Christmas)</li>
</ul>
<h2>Objectives:</h2>
<p>Using Python and Quarto, we will:</p>
<ul>
<li>Analyze data across various time periods and locations</li>
<li>Visualize findings with meaningful dashboards</li>
<li>Gain insights for better retail strategy planning</li>
</ul>
<h2>Tools and Technologies</h2>
<ul>
<li><strong>Python</strong>: Primary language for data manipulation and analysis.</li>
<li><strong>Quarto</strong>: For creating shareable reports with combined code, text, and visualizations.</li>
<li><strong>Pandas</strong>: For efficient data handling and cleaning.</li>
<li><strong>Matplotlib/Seaborn</strong>: Libraries used for creating sales trend visualizations.</li>
<li><strong>Jupyter Notebook</strong>: The interactive environment for data analysis.</li>
</ul>
<h2>Project Structure</h2>
<ul>
<li><strong>Data Preparation</strong>: Cleaning, formatting, and organizing sales data.</li>
<li><strong>Exploratory Data Analysis (EDA)</strong>: Discovering patterns and trends.</li>
<li><strong>Report Generation</strong>: Documenting findings using Quarto for a shareable report.</li>
<li><strong>Dashboard Development</strong>: Prototyping visualizations for store-level, district-level, and new store analyses.</li>
</ul>
<h2>Dashboard Prototype</h2>
<ol>
<li><strong>Store Sales Overview</strong><ul>
<li><strong>Purpose</strong>: To provide a summary of store-level performance, including total sales, variance, and average sales per unit.</li>
<li><strong>Visuals</strong>: Sales distributions, comparisons, and sales per store type.</li></ul></li>
<li><strong>District Monthly Sales</strong><ul>
<li><strong>Purpose</strong>: To analyze sales at the district level, focusing on month-over-month changes and variances.</li>
<li><strong>Visuals</strong>: Monthly sales breakdown, trend analysis, and variance charts by district.</li></ul></li>
<li><strong>New Stores Analysis</strong><ul>
<li><strong>Purpose</strong>: To examine the performance of newly opened stores, covering metrics like sales per square foot and geographic distribution.</li>
<li><strong>Visuals</strong>: Map visualizations, monthly sales, and sales by store area.</li></ul></li>
</ol>
<h2>Data Dictionary and Key Measures</h2>
<p>This project uses a retail dataset with key fields such as <code>DistrictID</code>, <code>ItemID</code>, <code>LocationID</code>, <code>OpenDate</code>, and various sales metrics. These fields enable analysis across multiple dimensions, such as:</p>
<ul>
<li><strong>Markdown and Regular Sales</strong>: Summed up by category.</li>
<li><strong>Total Sales</strong>: Combination of Markdown and Regular Sales.</li>
<li><strong>Sales Variance</strong>: Month-over-month and year-over-year changes.</li>
<li><strong>Sales per Square Foot</strong>: Efficiency metric calculated by dividing total sales by store area.</li>
</ul>
<h2>Challenges and Learnings</h2>
<p>Key challenges included:</p>
<ul>
<li>Creating accurate data visualizations and aligning time-series data</li>
<li>Managing store openings chronologically</li>
<li>Calculating district-level trends</li>
</ul>
<p>Overcoming these challenges improved data management and visualization skills and deepened understanding of retail sales analytics.</p>
